PROJECTS

While working at The Learning Corp, I worked with another co-op to automate the process of creating builds of the iOS and Android apps by implementing a continuous integration and continuous delivery system. Once a pull request on GitHub is merged into the master branch of the code, the pipeline automatically builds, tests and deploys the app to firebase, a platform through which other employees can download and test the app with their own devices. It also generates reports on the style and quality of the code using swiftlint and sonarqube, and notifies developers via Slack when new builds are created. The pipleines which run through these steps was created using Groovy.

Ultimately, this project was able to create builds without any need for developers to intervene -- a process which used to take them about two hours per build -- leaving them more time to create features. The ability to create builds every time new code was implemented in the code base, as opposed to every three days or so, also helped to make the QA process more efficient and accurate.

I collaborated with another student to develop a machine learning model that could take a short phrase and determine whether or not it was funny. Working with a dataset from Reddit containing thousands of jokes and a rating on how funny they were, we used Python’s Natural Language Toolkit to extract “interesting” words, which became the features of our machine learning model. We then implemented the Naive Bayes, Decision Tree and Random Forest algorithms and compared them to see which to see which was most suited to our problem.

For my Computer Communications and Networks class, I implemented and evaluated three different protocols (Stop-and-Wait, Go Back N and Selective Repeat) for achieving reliable data transfer over the UDP transport protocol using Java. I created a sender and receiver and provided the functionality to transfer a large file (specified by the user) between them over a link as a sequence of smaller messages.

Because of the nature of UDP, the sender needed to add a header to each packet it sent, indicating the order of the packets. The receiver then used this information to piece the file back together, and to send an acknowledgment to the sender validating that each packet was received.

WIT Social Media - In high school I was the only female in my computer science class of 30 students. At my university, less than 25% of enrolled computer science majors are female, and that number is sadly one of the highest in the nation. I spent much of my last co-op being the only woman on the entire engineering team. Although I am so passionate about computer science, it can sometimes become exhausting feeling as if I constantly have to prove myself and represent my entire gender. Thankfully I have found a close group of female friends pursuing technology that I can relate to and rely on, and their support has been invaluable to my growth as a software engineer. That is why a friend and I decided to try and create a social media website, specifically for women and non-binary individuals, to be able to expand their networks and have a supportive environment where they can interact with other female computer scientists, outside of the male dominated communities that we usually find ourselves in. This project is in the early stages, but we are currently learning Ruby on Rails, which we hope to use to code our website!
